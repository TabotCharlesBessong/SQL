●The index file for a primary index occupies a much smaller space than does the data file
–Fewer index entries than there are records in the data file
–Each index entry is typically smaller in size than a data record because it has only two 
fields
●Consequently, more index entries than data records can fit in one block
●Hence, a binary search on the index file requires fewer block accesses than a binary 
search on the data file
–For an ordered data file with b blocks, log 2b block accesses
–For primary index file with only b i blocks
○To locate a record with a search key value requires a binary search of that index, 
and then access to the block containing that record: a total of log 2bi + 1 accesses26/04/24 Denis L. Nkweteyim@UB - IST608 (Advanced Topics in DB Management) 11Primary Indexes

●A binary search requires approximately (log 2bi) block accesses for an index

●Idea behind a multilevel index
–Reduce the part of the index that we continue to search by bfr i, the blocking factor for the index
–This way, the search space is reduced much faster
–bfri is called the fan-out of the multilevel index (we will refer to it by the symbol fo)
●Note
–Binary search divides the record search space into two halves at each step of the search
–Multilevel search divides the search space n-ways (where n = the fan-out) at each search step 
using the multilevel index
–Searching a multilevel index requires approximately (log fobi) block accesses
○This is a substantially smaller number than for a binary search if the fan-out is larger than 2

●Search tree
–Special type of tree that is used to guide the search for a record, given

–Multilevel indexes we saw earlier is a variation of a search tree

●Algorithms are necessary for inserting and deleting search values into and from the tree
●These algorithms do not guarantee that a search tree is balanced (by having leaf nodes at the same 
level)
●In a balanced tree
–Nodes are evenly distributed, so that the depth of the tree is minimized for the given set of keys 
and the tree does not get skewed with some nodes being at very deep levels
–Search speed is uniform (average time to find any random key is roughly the same)
●Another goal
–The tree should not need too much restructuring as records are inserted into and deleted from the 
main file
–Achievable by
○Making sure the nodes to be as full as possible
○Avoiding empty nodes when many deletions are made26/04/24 Denis L. Nkweteyim@UB - IST608 (Adv
