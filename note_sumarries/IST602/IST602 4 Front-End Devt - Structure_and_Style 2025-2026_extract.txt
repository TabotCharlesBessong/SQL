 4.1 Cascading Style Sheets (CSS)
 4.1.1 Intro to CSS
• HTML was designed to specify the semantics (i.e., content and structure) of a document
◦ e.g., which parts are elements of section headers, body text, links, lists, tables, etc.)
◦ But HTML markup can also be used to indicate not only semantics of a document, but 
also some presentation (fonts, spacing, sizes, colours, positioning, etc.)
◦ It is  better though, not to mix presentation with content
▪ Use markup predominantly for indicating the semantics of a document and use a 
separate mechanism to determine exactly how information contained in the 
document should be presented
• CSS provide a mechanism to separate semantics form presentation
• This separation of structure from presentation simplifies maintaining and modifying web 
pages, especially on large-scale websites
◦ If a website’s presentation is determined entirely by a style sheet, you can simply swap 
in a new style sheet to completely change the site’s appearance
 4.1.2 CSS Rule (Syntax)
• Comprises
◦ A selector
◦ The HTML element you want to style (or format)
• A declaration block
◦ One or more declarations separated by semicolons
◦ Each declaration includes a property name and a value, separated by a colon
◦ Declaration always ends with a semicolon, and declaration groups are surrounded by 
curly braces

• Three approaches
◦ Inline styles
◦ Embedded style sheets

 4.1.3.1 Inline Styles
• Declare an individual element’s format using the HTML5 style attribute
• Override any other styles applied using embedded styles sheets and external style sheets
• Note: Inline styles do not truly separate presentation from content

 4.1.3.3 External style sheets
• Separate document that contain only CSS rules
• Enables us provide a uniform look and feel to an entire website (or to a portion of one)
• Can reuse the same external style sheet across multiple websites
• Different pages on a site can all use the same style sheet
• When changes to the styles are required, you need to modify only a single CSS file to make 
style changes across all the pages that use those styles

 4.2 Introduction to Bootstrap for responsive front-end design
See https://www.w3schools.com/bootstrap5/bootstrap_get_started.php
What is Bootstrap?
• An open-source front-end framework used to build responsive, mobile-first web applications
• Provides a collection of pre-designed HTML and CSS templates for interface elements like 
typography, forms, buttons, and navigation, along with optional JavaScript plugins for 
interactive features. 
• Core Features
◦ Responsive Grid System: Uses a 12-column flexbox-based grid that automatically 
reorders or resizes elements to fit desktops, tablets, and smartphones
◦ Pre-built Components: Includes ready-to-use UI elements such as
▪ Navigation: Navbars, breadcrumbs, and pagination
▪ Forms: Styled input fields, checkboxes, and validation features
▪ Interactivity: Modals, carousels, tooltips, and dropdowns powered by JavaScript
◦ Mobile-First Approach: Styles are optimized for mobile devices by default and scaled up 
for larger screens using CSS media queries
Getting Bootstrap
• Either download from getbootstrap.com or include it from a CDN (Content Delivery 
Network) like jsDelivr
• Including from a CDN is most common usage. Why?
◦ Many users already have downloaded Bootstrap 5 from jsDelivr when visiting another 
site

faster loading time
◦ Also, most CDNs will make sure that once a user requests a file from it, it will be served 
from the server closest to them, which also leads to faster loading time
• How to include Bootstrap 5?
Key steps in using Bootstrap
• Add the HTML5 doctype
• Mobile-first
◦ To ensure proper rendering and touch zooming, add the following <meta> tag inside the 
<head> element
◦ width=device-width: Sets the width of the page to follow the screen-width of the device 
(which will vary depending on the device)
◦ initial-scale=1: sets the initial zoom level when the page is first loaded by the browser
• Containers
◦ A containing element is used to wrap site contents
◦ 2 container classes to choose from
▪ .container class provides a responsive fixed width container
▪ .container-fluid class provides a full width container, spanning the entire width of the 
viewport
