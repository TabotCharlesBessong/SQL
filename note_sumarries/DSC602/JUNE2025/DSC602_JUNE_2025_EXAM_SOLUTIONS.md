### Q1.1: What is Meant by Information Visualization? (6 marks - 3+3)

#### PART A: Definition (3 marks)

**Information Visualization (InfoVis) Definition:**

Information Visualization is the discipline of representing abstract, non-spatial data in visual and interactive forms to support human perception, cognition, and decision-making. It transforms data that lacks inherent spatial dimensions into visual representations that humans can readily interpret.

**Key Components:**
1. **Input**: Abstract data with no inherent 2D/3D structure
   - Examples: Financial data, text documents, relationships, hierarchies
   - NOT naturally spatial (unlike maps or scientific data)

2. **Process**: Transformation through visual encoding
   - Map data attributes to visual properties (color, size, position, shape)
   - Create perceptually effective representations
   - Enable interactive exploration

3. **Output**: Visual/interactive representations
   - Enables pattern recognition
   - Supports analytical reasoning
   - Facilitates insight generation
   - Communicates findings effectively

**Formal Definition (from Lectures):**
"The use of computer-supported, interactive, visual representations of abstract data to amplify cognition and enable new discoveries."

**Why Separate from General Visualization:**
- **Scope**: Focuses on abstract information (not naturally spatial)
- **Goal**: Enable analytical discovery (not just display)
- **Interaction**: Heavy emphasis on interactivity and exploration
- **Complexity**: Handles highly multi-dimensional data

#### GRAPH 1: LINE GRAPH (2 marks)

**Description** (1 mark):
A line graph displays data points connected by lines, showing change over time or ordered categories.
- **X-axis**: Time (years, months, days) or ordered categories
- **Y-axis**: Quantitative values (sales, temperature, population)
- **Marks**: Points connected by lines
- **Channels**: Position (x,y), color (multiple series), line style (trend types)

**Visual Example**:
```
Sales (Y)
   |     /\      /\
   |    /  \    /  \
   |   /    \  /    \
   |  /      \/      \
   |_____________________ Time (X)
   Jan  Feb  Mar  Apr  May
```

**Why Useful** (1 mark):
- **Temporal Trends**: Instantly shows if data is increasing, decreasing, or stable
- **Pattern Recognition**: Reveals cycles, seasonality, and anomalies
- **Prediction**: Observers can extrapolate future trends
- **Multiple Series**: Can compare different time series on same graph (e.g., sales vs. costs)
- **Intuitive**: Most familiar to general audiences
- **Efficient**: Shows 2 dimensions clearly on standard 2D plot

**Business Examples**:
- Stock prices over time (detect market trends)
- Website traffic over months (see seasonality)
- Temperature trends (climate monitoring)
- Patient recovery progress (healthcare)

A bar chart displays values as rectangular bars, with height (or length) representing magnitude.

- **Y-axis**: Quantitative values
- **Marks**: Rectangles (bars)

**Variants**:
- Horizontal bars (better for long category names)

#### DATA STRUCTURE 1: DICTIONARY (3 marks)

**Definition** (1 mark):
A Dictionary is an unordered collection of key-value pairs enclosed in curly braces `{}`. Each unique key maps to exactly one value.

**Syntax**:
```python
# Basic dictionary
person = {'name': 'John', 'age': 30, 'city': 'London', 'salary': 50000}

# Dictionary with mixed types
data = {'employees': 100, 'revenue': 1000000.50, 'status': True, 'departments': ['IT', 'HR', 'Sales']}
```

**Key Characteristics** (1 mark):
- **Unordered**: No guarantee of element order (Python 3.7+ maintains insertion order, but not logically important)
- **Mutable**: Can be modified (add, remove, update items)
- **Key-Value Pairs**: Each element is a unique key linked to a value
- **Flexible Types**: Keys and values can be different types (though keys usually strings/integers)
- **Access by Key**: Retrieve values using `dictionary[key]`, not by position

**Data Visualization Application** (1 mark):
**Metadata and Configuration**:
```python
# Configuration for visualization
config = {
    'plot_type': 'bar',
    'colors': ['red', 'blue', 'green'],
    'title': 'Sales by Region',
    'xlabel': 'Region',
    'ylabel': 'Sales (Million $)',
    'figure_size': (12, 6)
}

# Data with labels
country_data = {
    'France': 85,
    'Germany': 95,
    'Italy': 62,
    'Spain': 48,
    'Portugal': 32
}

# Visualization loop
for country, gdp in country_data.items():
    print(f"{country}: {gdp}B EUR")
```

**Why Important for Visualization**:
- Store and pass visualization parameters
- Map category names to values
- Organize multi-dimensional data
- Flexible key-based access (more intuitive than numeric indices)

#### DATA STRUCTURE 2: TUPLE (3 marks)

**Definition** (1 mark):
A Tuple is an ordered, immutable collection of elements enclosed in parentheses `()`. Once created, it cannot be modified (no adding, removing, or changing elements).

**Syntax**:
```python
# Basic tuple
coordinates = (x, y)
rgb_color = (255, 128, 0)
point_in_3d = (10.5, 20.3, 15.7)

# Unpacking tuple
x, y = coordinates
r, g, b = rgb_color
```

**Key Characteristics** (1 mark):
- **Ordered**: Elements have fixed positions (index 0, 1, 2, ...)
- **Immutable**: Cannot be changed after creation
- **Indexed Access**: Retrieve by position: `tuple[0]`, `tuple[1]`
- **Lightweight**: More memory efficient than lists
- **Hashable**: Can be used as dictionary keys (unlike lists)
- **Unpacking**: Can assign multiple values: `a, b, c = tuple_of_3`

**Data Visualization Application** (1 mark):
**Coordinate Storage and Graphical Properties**:
```python
# Store 2D/3D coordinates
points = [
    (10, 20),    # (x, y)
    (15, 25),
    (20, 30)
]

# Store colors as RGB tuples
colors = [
    (255, 0, 0),      # Red
    (0, 255, 0),      # Green
    (0, 0, 255)       # Blue
]

# Store dimension sizes
figure_size = (12, 8)   # (width, height)
position = (100, 150)   # (x_pixel, y_pixel)

# Return multiple values from function
def get_plot_range(data):
    return (min(data), max(data))  # Returns tuple

min_val, max_val = get_plot_range([1, 5, 3, 8, 2])
```

**Why Important for Visualization**:
- Store fixed coordinate pairs (efficient memory)
- Immutability ensures data integrity
- Represent colors (RGB values don't change)
- Return multiple values from functions
- Use as dictionary keys for unique visual properties

#### DATA STRUCTURE 3: SERIES (3 marks)

**Definition** (1 mark):
A Pandas Series is a 1-dimensional labeled array that contains data of any type and an associated axis labels (index). It's built on NumPy and designed for data manipulation and analysis.

**Syntax**:
```python
import pandas as pd

# From list
ages = pd.Series([25, 30, 35, 40, 45], index=['Alice', 'Bob', 'Carol', 'David', 'Eve'])

# From dictionary
sales = pd.Series({
    'Q1': 100000,
    'Q2': 150000,
    'Q3': 120000,
    'Q4': 180000
})

# Access elements
sales['Q1']          # 100000
sales[0]             # 100000 (first element)
sales[['Q1', 'Q3']]  # Multiple elements
```

**Key Characteristics** (1 mark):
- **Labeled**: Each value has an associated index label (not just numeric position)
- **Homogeneous**: All elements typically same data type (can mix, but not recommended)
- **Mutable**: Can be modified (add, remove, change values)
- **Operations**: Supports mathematical operations, filtering, aggregation
- **Integration**: Works seamlessly with DataFrames and visualization libraries
- **Efficient**: Optimized for data operations using NumPy under the hood

**Data Visualization Application** (1 mark):
**Core Data for Visualization**:
```python
# Sales data
sales_series = pd.Series(
    [100000, 150000, 120000, 180000],
    index=['Q1', 'Q2', 'Q3', 'Q4'],
    name='Quarterly Sales'
)

# Visualization
import matplotlib.pyplot as plt
sales_series.plot(kind='bar', title='Sales by Quarter')
plt.ylabel('Sales ($)')
plt.show()

# Regional data
regional_sales = pd.Series({
    'North': 500000,
    'South': 350000,
    'East': 600000,
    'West': 450000
})

# Filtering for visualization
high_sales = regional_sales[regional_sales > 400000]
# Use high_sales for visualization

# Time series
import numpy as np
dates = pd.date_range('2025-01-01', periods=100)
values = np.random.randn(100).cumsum()
time_series = pd.Series(values, index=dates)
time_series.plot()  # Automatic time axis formatting
```

**Why Important for Visualization**:
- Direct integration with matplotlib (`.plot()` method)
- Labeled indices provide semantic meaning
- Filtering and transformation before visualization
- Time series support with date indices
- Built-in statistical methods for exploratory analysis
- Efficient memory usage for large datasets

### Q2.3: Data Frame Particularity for Efficient Programming (5 marks - 3+2)

#### What Is a Data Frame? (3 marks)

**Definition** (1 mark):
A DataFrame is a 2-dimensional, labeled data structure with rows and columns, where each column is a Series (1D array of data). It's the primary data structure in pandas for data manipulation and analysis.

**Structure Visualization**:
```
DataFrame Structure:
        Name   Age  Salary  Department
    0   Alice   30  50000   IT
    1   Bob     25  45000   HR
    2   Carol   35  60000   IT
    3   David   28  52000   Sales

Rows (Index):     0, 1, 2, 3 (labels each row)
Columns:          Name, Age, Salary, Department (labeled each column)
Data Types:       object, int64, int64, object
```

**Key Characteristics** (1 mark):
- **2D Structure**: Rows × Columns organization
- **Labeled**: Both row indices and column names (semantic meaning)
- **Heterogeneous**: Different columns can have different data types
- **Mutable**: Can add/remove/modify rows and columns
- **Indexed**: Supports both label-based and position-based access
- **Operations**: Built-in methods for filtering, grouping, aggregation, transformation

**Syntax**:
```python
import pandas as pd

# Create DataFrame from dictionary
data = {
    'Name': ['Alice', 'Bob', 'Carol', 'David'],
    'Age': [30, 25, 35, 28],
    'Salary': [50000, 45000, 60000, 52000],
    'Department': ['IT', 'HR', 'IT', 'Sales']
}
df = pd.DataFrame(data)

# Access data
df['Name']         # Column as Series
df.loc[0]          # Row 0 as Series
df.iloc[0, 0]      # Element at row 0, col 0
df[df['Age'] > 28] # Filter rows
```

---

#### Why DataFrames Are Particularly Efficient for Programming (2 marks)

**Efficiency Aspect 1: Vectorized Operations** (1 mark):
Traditional programming requires loops:
```python
# INEFFICIENT: Loop over each row
salaries = [50000, 45000, 60000, 52000]
bonuses = []
for salary in salaries:
    bonuses.append(salary * 0.1)
```

With DataFrames:
```python
# EFFICIENT: Vectorized operation
df['Bonus'] = df['Salary'] * 0.1  # One-liner!
# Internally optimized in NumPy/C, much faster
```

**Benefits**:
- Dramatically faster for large datasets (100x+ speedup)
- Cleaner, more readable code
- Automatic memory optimization
- Parallelizable on multiple cores

**Efficiency Aspect 2: Built-in Methods Eliminate Code** (1 mark):
```python
# Complex analysis in one line
df.groupby('Department')['Salary'].mean()
# Instead of: manual looping, accumulating, dividing

# Filtering in one line
young_high_earners = df[(df['Age'] < 30) & (df['Salary'] > 48000)]
# Instead of: nested loops and conditional checks

# Time series operations
df.set_index('Date').resample('M').sum()
# Instead of: manual date parsing, grouping, aggregation

# Missing data handling
df.fillna(df.mean())
# Instead of: iterating, identifying, replacing manually
```

**Value for Data Visualization Programming**:
- Prepare data in few lines
- Filter/aggregate before plotting
- Transform columns easily
- Combine multiple datasets efficiently
- Direct plotting integration: `df.plot()`

## QUESTION 3: DATA VISUALIZATION GRAPHICS AND CHARACTERISTICS (25 marks)

### OVERVIEW
**Purpose**: Identify which type of graph is suitable for which context and justify answers.

**6 Graph Types to Analyze**: Line graph, Bar chart graph, Histogram graph, Flow chart graph, Box plot graph, Bubble map graph.

**Mark Breakdown**: 4 marks × 6 graphs + 1 bonus = 25 marks
- Each graph: 4 marks = (1 mark) definition + (2 marks) best use contexts + (1 mark) justification

---

#### GRAPH 1: LINE GRAPH (4 marks)

**Definition** (1 mark):
Line graph connects data points with lines, showing values on y-axis against time/sequence on x-axis. Primary use: showing trends, changes, and progression over time.

**Best Use Contexts** (2 marks):
1. **Temporal Trends** (1 mark):
   - Stock prices over months/years
   - Website traffic by day
   - Patient health metrics over treatment period
   - Economic indicators (GDP, inflation) over decades

2. **Comparison of Multiple Series Over Time** (1 mark):
   - Revenue vs. Expenses over quarters
   - Different product sales trends on same chart
   - Temperature highs vs. lows by month
   - Multiple competitor performance tracking

**Justification** (1 mark):
- **Pattern Recognition**: Human eye instantly detects trends (rising, falling, stable)
- **Seasonality**: Cyclical patterns become obvious
- **Anomaly Detection**: Sudden spikes/drops catch attention
- **Prediction**: Users can extrapolate future values
- **Multiple Series**: Different colored lines enable comparison
- **Efficient**: 2 dimensions (time × value) clearly displayed
- **Intuitive**: Most familiar to general audiences

---

#### GRAPH 2: BAR CHART GRAPH (4 marks)

**Definition** (1 mark):
Bar chart displays categorical data as rectangular bars with heights proportional to values. Bars can be vertical (column chart) or horizontal. Enables direct magnitude comparison across categories.

**Best Use Contexts** (2 marks):
1. **Categorical Comparison** (1 mark):
   - Sales by region/country
   - Market share by competitor
   - Preferred ice cream flavors (survey)
   - Employee satisfaction by department
   - Product revenue ranking

2. **Part-to-Whole Relationships** (1 mark) - *Stacked bar variant*:
   - Company revenue breakdown by division
   - Website traffic sources (organic, paid, direct, referral)
   - Customer demographics (age groups)
   - Budget allocation across departments

**Justification** (1 mark):
- **Precise Comparison**: Bar length directly proportional to value (accurate perception)
- **Simplicity**: Easy for any audience to understand
- **Flexibility**: Works with numerous categories without clutter
- **Composition Analysis**: Stacked variant shows both totals and breakdown
- **Value Labels**: Can add exact numbers on bars
- **Cultural Familiarity**: Most common graph type in business/media

---

#### GRAPH 3: HISTOGRAM GRAPH (4 marks)

**Definition** (1 mark):
Histogram shows distribution of continuous data using adjacent bars (bins), where height represents frequency. X-axis represents data ranges; Y-axis represents count/frequency.

**Best Use Contexts** (2 marks):
1. **Distribution Analysis** (1 mark):
   - Customer age distribution
   - Income levels in population
   - Test score distribution in class
   - Product defect rates
   - Response times in call center

2. **Outlier and Spread Detection** (1 mark):
   - Quality control: Identify out-of-specification items
   - Medical screening: Detect abnormal values
   - Statistical analysis: Check normality assumption
   - Data cleaning: Identify data entry errors

**Justification** (1 mark):
- **Distribution Shape**: Instantly shows if data is normal, skewed, bimodal
- **Central Tendency**: Where most data concentrates (mode)
- **Spread**: How variable the data is
- **Outliers**: Extreme values visible as isolated bars
- **Data Quality**: Suspicious patterns reveal problems
- **Statistical Foundation**: Basis for further analysis (normal distribution assumptions, etc.)

---

#### GRAPH 4: FLOW CHART GRAPH (4 marks)

**Definition** (1 mark):
Flow chart is a diagram showing processes, decision points, and data flow using connected boxes, diamonds, and arrows. Represents workflows, algorithms, or systems step-by-step.

**Best Use Contexts** (2 marks):
1. **Process Documentation** (1 mark):
   - Software development workflows
   - Customer service procedures
   - Manufacturing processes
   - Loan approval process
   - Customer journey mapping

2. **Decision Trees and Logic** (1 mark):
   - Diagnostic procedures (medical decision tree)
   - Troubleshooting guides
   - Business rules engine
   - Algorithm flowchart
   - Quality assurance testing procedures

**Justification** (1 mark):
- **Clarity**: Sequential steps instantly understood
- **Decision Visualization**: Shows branching logic (yes/no, pass/fail)
- **Communication**: Non-technical people understand processes
- **Optimization**: Identifies inefficiencies and bottlenecks
- **Training**: Helps new employees learn procedures
- **Documentation**: Serves as official reference material

---

#### GRAPH 5: BOX PLOT GRAPH (4 marks)

**Definition** (1 mark):
Box plot (box-and-whisker) displays data distribution using five-number summary: minimum, Q1 (25th percentile), median (Q2), Q3 (75th percentile), maximum. Shows spread, skewness, and outliers.

**Visual Structure**:
```
Box Plot Components:
        *           ← Outlier (outside 1.5×IQR)
        |
     ───┬───       ← Whisker (max non-outlier)
        │ │
        │ │        ← Box (Q1 to Q3)
    ─── ─ ───      ← Line in box (median)
        │ │
        │ │
     ───┴───       ← Whisker (min non-outlier)
```

**Best Use Contexts** (2 marks):
1. **Distribution Comparison Across Groups** (1 mark):
   - Salary distribution by department
   - Test scores by school
   - House prices by neighborhood
   - Response times by service tier
   - Production times by manufacturing plant

2. **Outlier Identification** (1 mark):
   - Detect unusual measurements
   - Identify exceptional cases for investigation
   - Quality control: Flag defective units
   - Fraud detection: Unusual transactions

**Justification** (1 mark):
- **Compact**: Shows entire distribution in small space
- **Multiple Comparisons**: Side-by-side boxes compare groups
- **Statistical Summary**: Displays key quantiles and spread
- **Outlier Visibility**: Automatically identifies and marks anomalies
- **Skewness Detection**: Asymmetric boxes show distribution bias
- **Professional**: Standard in scientific/statistical reporting

---

#### GRAPH 6: BUBBLE MAP GRAPH (4 marks)

**Definition** (1 mark):
Bubble map is a geographic visualization overlaying bubbles (circles) on a map, where bubble size represents magnitude and color represents categorical variable. Combines geographic position with quantitative data.

**Visual Example**:
```
Map with Bubbles:

         ◯ Large bubble (high value)
              
    ◯ Medium        ◯ Large
    
         ◯ Small (low value)

Legend: Size = Sales Volume, Color = Region/Category
```

**Best Use Contexts** (2 marks):
1. **Geographic Distribution Analysis** (1 mark):
   - Sales by city/region
   - Population density by country
   - Store locations and performance
   - Disease outbreaks by location
   - Weather data by geographic point

2. **Multi-Dimensional Geographic Analysis** (1 mark):
   - Sales (bubble size) by region (position) by product (color)
   - Population (size) by country (location) by income level (color)
   - Store performance (size) by location (position) by category (color)

**Justification** (1 mark):
- **Geographic Context**: Location provides immediate spatial understanding
- **Multiple Dimensions**: Size + color + position = 3+ data dimensions
- **Pattern Discovery**: Clusters and gaps visible spatially
- **Intuitive**: Most people understand maps
- **Business Value**: Sales/performance in context of physical locations
- **Engagement**: Interactive maps are compelling for presentations
