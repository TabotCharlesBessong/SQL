## Key Concepts

### 1. Data Flow Diagrams (DFD)
- Components: Processes, Data Stores, External Entities, Data Flows
- Types: Context Diagram (Level 0), Level 1, Level 2, etc.
- Notation and symbols

### 2. Activity Diagrams
- Swim lanes for organizational structure
- Process representation
- Flow of activities

### 3. Requirements Engineering
- Functional requirements
- Non-functional requirements
- Requirements prioritization techniques (MoSCoW, Kano Model, etc.)

### 4. System Design
- Feasibility studies
- GUI design principles
- Error minimization strategies

### 5. Database Design
- Entity Relationship Diagrams (ERD)
- Relationship types and cardinalities
